## Nuevas Funcionalidades

- **Soporte para EKS 1.33:** SleakOps ahora soporta upgrades de clusters a EKS 1.33, incluyendo actualizaciones de `code-dns`, `kubeproxy` y AMIs de NodePool (tanto Bottlerocket como AL2023).
- **Upgrade de Karpenter a v1.8:** Se actualizó Karpenter de la versión v1.3.2 a v1.8, mejorando el rendimiento y la estabilidad del autoescalado de nodos.
- **Dashboard de Monitoreo de Karpenter:** Se agregó un dashboard dedicado de Karpenter en Grafana, proporcionando visibilidad sobre el comportamiento y las m\u00e9tricas del autoescalado de nodos.
- **Menu Simplificado:** Se reorganizó el sidebar.
- **Nombre al Clonar Environment:** Al clonar un Environment, ahora se genera automáticamente un nombre modificado para evitar conflictos de nombres con el original.
- **Edición de VariableGroup tipo File:** Se mejoró la experiencia de edición de VariableGroups de tipo `file`, ofreciendo un flujo de trabajo más intuitivo.
- **Mejoras de navegación entre Formularios:** Cancelar un formulario ahora redirige al usuario a la página anterior en lugar de volver a la vista de lista del recurso por defecto.
- **Tooltip de Próxima Ejecución de CronJob:** El tooltip de los CronJobs ahora muestra la próxima fecha y hora de ejecución programada.
- **Delegación de Alias DNS por Registro A:** Se agregó soporte para la delegación de alias de dominio mediante registros DNS de tipo A, permitiendo a los usuarios con dominios gestionados fuera de SleakOps delegarlos correctamente.
- **CPU Requests en Prometheus:** Los usuarios ahora pueden definir valores de `request_cpu` en Prometheus junto con los existentes `backend_memory_requests` para un mejor control de la asignación de recursos.
- **Retención de Métricas en Grafana:** Se aumentó el tiempo de retención de métricas en Grafana para mayor disponibilidad de datos históricos.
- **Mejoras en Reportes de Downtime:** Se mejoró la estructura y el logging del ServiceDowntimeReport para un seguimiento más preciso de caídas de servicio de fracciones de segundo.

## Correcciones de Errores

- **Visualización de Tiempo Programado:** Se corrigió un problema donde el tiempo programado no era visible en los clusters de producción debido a una consulta incorrecta.
- **Selección de Projects en VariableGroup:** Se corrigió la selección de Projects para la replicación de VariableGroups, ahora muestra correctamente solo los Projects que comparten el mismo Cluster.
- **Error Vertical de VariableGroup:** Se resolvió un error de visualizaci\u00f3n que ocurr\u00eda al pegar VariableGroups en modo texto.
- **Validación de VariableGroup ENV:** Se corrigi\u00f3 la validaci\u00f3n para VariableGroups de tipo ENV en modo texto para prevenir env\u00edos con contenido inv\u00e1lido.
- **Contenido de Archivo en VariableGroup:** Se corrigieron problemas de visualizaci\u00f3n con contenido de archivo largo en el detalle de VariableGroup.
- **Eliminación de VariableGroup Replicada:** Eliminar un VariableGroup replicado ahora ejecuta correctamente los deploys en todos los Projects asociados, no solo en el principal.
- **Pegado en Modo Texto de VariableGroup:** Se corrigi\u00f3 la capacidad de pegar listas grandes de VariableGroups en modo texto para gesti\u00f3n de configuraciones masivas.
- **Visualización de Notificaciones:** Se corrigieron problemas en la visualizaci\u00f3n y el hover de las notificaciones.
- **Navegación al Cancelar en Project:** Se corrigi\u00f3 el comportamiento de navegaci\u00f3n al cancelar en un formulario de Project.
- **Fecha en Monitoring Dependency:** Se corrigi\u00f3 la visualizaci\u00f3n del selector de fecha en el m\u00f3dulo de Monitoring Dependency para un an\u00e1lisis de m\u00e9tricas adecuado.
- **Error al Cambiar Dominio:** Se resolvi\u00f3 un error 400 al cambiar dominios en un Environment, asegurando la correcta actualizaci\u00f3n de URLs y creaci\u00f3n de releases.
- **Patch de Cluster con Deletion Protection:** La configuraci\u00f3n del cluster ahora puede editarse incluso cuando una dependencia RDS tiene `deletion_protection` habilitado.
- **Campos de Custom Ingress:** Se eliminaron asteriscos incorrectos de campos opcionales en el formulario de Custom Ingress Configuration.
- **Navegaci\u00f3n de P\u00e1ginas y Padding:** Se corrigi\u00f3 el flujo de navegaci\u00f3n de p\u00e1ginas y se ajust\u00f3 el padding para una mejor consistencia de dise\u00f1o.
- **Contraseña de Usuario IAM:** Se corrigi\u00f3 un problema con las contrase\u00f1as autogeneradas al crear usuarios de IAM.
- **Deploy de ChartDependency:** Crear o editar ChartDependencies con `deploy=true` ahora ejecuta correctamente el proceso de deploy.
- **Optimización de Instancias en NodePool:** Se corrigi\u00f3 un problema donde los NodePools creaban instancias `t3a.micro` innecesarias, mejorando la eficiencia en la asignaci\u00f3n de recursos.
- **Mensajes de Error de Pulumi:** La captura de errores de Pulumi ahora muestra el error real de AWS/Pulumi en lugar del mensaje gen\u00e9rico "Resource monitor has terminated".
- **Validación de Tarea de Update CNI:** Se agreg\u00f3 validaci\u00f3n para prevenir la ejecuci\u00f3n simult\u00e1nea de tareas duplicadas de actualizaci\u00f3n de CNI.
