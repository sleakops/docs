## 🚀 New Features

- **Feature Presenter Modal**  
A new feature presenter modal now informs users about important product updates with images and descriptions when logging in. It only appears for unseen features and includes navigation between updates and Google Analytics tracking for engagement.

- **Dedicated Activity Log Screen**  
Activity logs now have their own dedicated screen, making it easier to review system events with improved visibility and navigation.

- **Cluster Addons Screen**  
Addons in clusters now display in a full screen table view instead of a popover, enabling better scaling for large numbers of addons and more consistent navigation. The table is accessible directly from the cluster detail and allows opening addon details.

- **Advanced Cost Monitoring and Limit Configuration**  
SleakOps now enables tracking AWS Bedrock request costs per client and supports defining a maximum cost limit via a dedicated section within Settings > IA Cost.

- **Infrastructure-Only Mode in Dockertron**  
Dockertron can now run in infrastructure-only mode, allowing projects with existing Dockerfiles to leverage SleakOps infrastructure modeling without regenerating Dockerfiles or docker-compose files.

- **Buildpacks Build Option at Project Level**  
Projects can now choose Buildpacks as their build method in addition to the default Docker, aligning with platforms like Northflank and Heroku.

- **Dependency Redis Cluster Support**  
SleakOps now supports the Redis dependency in cluster mode, enabling enhanced scalability and high availability for Redis workloads.

- **Detail Drawer for Members, Environments, and Clusters**  
You can now open a detail drawer from Members, Environments, and Clusters listings to view more information without leaving the list, improving navigation and resource management consistency.

---

## ✨ Improvements

- **Enhanced Table Usability**  
The Actions column is now fixed in all tables, staying visible during horizontal scrolling for quicker access to resource actions.

- **Improved Members Table**  
The Members table now displays user names above their email, visual access card icons, and role badges for clearer user identification and easier account management.

- **Addon Icon Display on Cluster Cards**  
Cluster cards now show the icons of installed addons for immediate visual context.

- **Cost Allocation Tag Activation for Bedrock**  
Bedrock request metadata tags are now automatically activated as AWS Cost Allocation Tags, enabling filtering in AWS Cost Explorer and improving cost tracking accuracy.

- **Better UI Details and Filtering**  
ContentType and DependencyType filters in the UI now display human-readable names instead of IDs. UI polish has also been applied for a more streamlined experience.

- **Custom Service Targeting in Workload Ingress**  
Ingress URLs and aliases for workloads can now be directed to specific Kubernetes services, and these assignments persist across redeploys for advanced routing scenarios.

- **Improved Webservice Form Usability**  
Webservice and workload forms are now organized into logical sections instead of a generic layout, offering both ease of use for beginners and advanced options for experts.

- **Environment Variable Group Filtering Enhanced**  
Filtering by project in variable groups now includes all variables directly linked to the project as well as shared variable groups.

---

## 🐛 Bug Fixes

- **Variable Groups Caching in Dependencies**  
Fixed an issue where variable groups in dependency details did not update correctly when switching between dependencies.

- **Volume Status Update Correction**  
Volumes now update their status properly after changes, ensuring reliable state tracking.

- **Activity Log Multi-line Display Fix**  
Fixed incorrect rendering of activity log entries containing multiple lines of text.

- **Broken Links in Deployments Detail Fixed**  
Resolved an issue where links in the Deployments Detail screen were not working for users.

- **Command Autocomplete Investigation**  
Investigated and addressed cases where commands autocompleted unexpectedly when cleared.
