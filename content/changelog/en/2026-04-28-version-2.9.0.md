## 🚀 New Features

- **CI/CD Configuration Generation for Projects**  
  Admins and editors can now automatically generate a pull request to configure SleakOps CI/CD in their project repositories directly from the UI or backoffice. This streamlines onboarding and eliminates manual setup steps.

- **Autodiagnostic Workflow Extended to Clusters, Builds, and Deployments**  
  The autodiagnosis feature is now available from the Clusters, Builds, and Deployments lists, allowing users to launch technical diagnostic tickets for these resources. This ensures faster troubleshooting and consistent diagnostic flows across all main resource types.

- **Cost Tracking for Bedrock API Usage**  
  Bedrock service usage is now tracked to calculate API request costs per customer account, making it easier to monitor and control expenses.

- **Cost, Security, and Infrastructure Reports via Chatbot**  
  The chatbot can now generate automated reports: cost breakdowns, security findings, and infrastructure diagrams. Results are visual and actionable, giving users and operators deeper insights from within the SleakOps console.

---

## ✨ Improvements

- **Pending Changes Workflow Centralization and Visibility**  
  The pending changes system for projects has been redesigned for better reliability and transparency. Changes are now accurately tracked and displayed, with improved support for multiple changes and environment isolation.

- **Enhanced Navigation: Consistent Titles on All Tabs**  
  All application tabs now display proper titles and resource names, making navigation clearer and more intuitive across projects, clusters, dependencies, and monitoring screens.

- **Dependency Monitoring: Title Improvements and Quick Access**  
  The dependency monitoring screen now shows the dependency name in the title and adds a button to open resource details in-place, improving resource identification and workflow consistency.

- **Autodiagnostic Cards Improvement**  
  The diagnostic cards now visually distinguish which resource each diagnosis belongs to, simplifying diagnosis review and matching.

- **Bedrock Anthropic Model Integration**  
  Bedrock Anthropic models are now enabled for programmatic use, thanks to the library update.

- **Boto3 Core Update**  
  The core modules now run on boto3 version 1.40+ to support more advanced Bedrock features, including programmatic integration with Claude.

- **Synchronization of Service States with Pending Approval Releases**  
  Service states (draft/published) are now kept in sync with their corresponding PendingApproval releases, leading to more accurate state representation.

---

## 🐛 Bug Fixes

- **Clusters and Environments Are Displayed Without an Account Selected**  
  Users can now view and manage Clusters and Environments without having an account selected, providing a true global infrastructure view and removing previous limitations.

- **Cluster Start/Stop Actions Work Without Account Selection**  
  The Start/Stop buttons for clusters now function correctly even when no account is selected, as the correct cluster account is automatically determined.

- **Activity Log: Correct Volume Icons and Tooltips**  
  The project volume icon and tooltip now correctly display in the Activity Log, improving resource identification.

- **Activity Log: Resolved Filters Rendering Issue**  
  Fixed a rendering bug that caused flickering or visual inconsistencies when applying filters in the Activity Log, resulting in a smoother and more stable experience.

- **Variable Group Secrets Display Bug**  
  Switching between secrets in different Variable Groups now correctly loads and displays each group's data, preventing sensitive information leaks between configurations.

- **BuildKit Builds with Long Resource Names**  
  Builds using long project, job, or service account names in BuildKit now process resource naming correctly and no longer get stuck, ensuring builds complete as expected.

- **Hover Remaining Time Calculation**  
  The remaining time calculation now updates automatically on each data fetch instead of only updating on hover, keeping times accurate in lists.
