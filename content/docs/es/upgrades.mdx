# Upgrades

Upgrades te permite programar y monitorear actualizaciones de versión de Kubernetes en todos tus entornos desde un solo lugar. Planificá cada upgrade con visibilidad completa de lo que va a cambiar, seguí su progreso en tiempo real y asegurate de que nada se ejecute en el momento equivocado ni pase desapercibido.

## Preguntas frecuentes

<details>
<summary>
### ¿Qué puedo actualizar?
</summary>
Desde la sección Upgrades podés programar y seguir actualizaciones de versión para:

- **Clusters** — versiones del control plane y los node groups de Kubernetes
- **Entornos** — actualizá todos los recursos asociados a un entorno en una secuencia coordinada
- **Add-ons** — mantené los add-ons del cluster (Grafana, Prometheus, KEDA, etc.) alineados con versiones compatibles

SleakOps valida la compatibilidad antes de programar el upgrade y marca cualquier bloqueador antes de que comience.
</details>

<details>
<summary>
### ¿Cómo programo un upgrade?
</summary>
1. Abrí **Upgrades** desde la barra lateral de SleakOps.
2. Seleccioná el cluster o entorno que querés actualizar.
3. Elegí la versión objetivo.
4. Seleccioná una fecha y ventana horaria para el upgrade.
5. Revisá el plan y confirmá.

SleakOps encola el upgrade y te envía una notificación cuando comienza y cuando finaliza.
</details>

<details>
<summary>
### ¿Qué sucede durante un upgrade?
</summary>
SleakOps actualiza los recursos en una secuencia controlada para minimizar el tiempo de inactividad:

1. Primero se actualiza el control plane.
2. Los node groups se actualizan de a uno — los workloads existentes son drenados y reprogramados antes de reemplazar cada nodo.
3. Los add-ons se actualizan a versiones compatibles con la nueva versión de Kubernetes.
4. El estado del upgrade se actualiza en tiempo real para que puedas seguir cada paso.

Si un paso falla, el upgrade se detiene y SleakOps muestra el error con contexto sobre qué hacer a continuación.
</details>

<details>
<summary>
### ¿Puedo actualizar múltiples entornos a la vez?
</summary>
Podés programar upgrades en múltiples entornos, pero SleakOps los ejecuta en secuencia por defecto para reducir el radio de impacto. Actualizá primero tu entorno de staging, validá el resultado y luego programá producción — este es el flujo recomendado para cambios críticos de infraestructura.
</details>

<details>
<summary>
### ¿Qué pasa si el upgrade falla?
</summary>
SleakOps detiene el upgrade en el primer paso fallido para evitar cambios en cascada. La consola muestra el error exacto, el recurso que falló y la acción recomendada.

Para upgrades de cluster, un reemplazo de nodo fallido no afecta los nodos ya actualizados — solo se pausa el node group en progreso, dejando el resto del cluster operativo.
</details>

<details>
<summary>
### ¿Puedo cancelar un upgrade programado?
</summary>
Sí. Los upgrades programados pueden cancelarse en cualquier momento antes de que comiencen. Una vez que un upgrade está en progreso no puede revertirse automáticamente — contactá a soporte si necesitás detener un upgrade activo.
</details>
