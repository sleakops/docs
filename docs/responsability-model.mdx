# Modelo de responsabilidad compartida. 


Usamos de manera transparente todos los servicios de AWS, por lo cual la responsabilidad se propaga a las normativas definidas por AWS.

Para resguardar y asegurarse de que su informacion no se pierda en un eventual problema con su servicio, es recomendable tener politicas de backups.
Por el momento no soportamos backups en nuestras dependencias (RDS, S3, RabbitMQ, etc.), para configurar esto, uds 
podria acceder a traves de su cliente AWS y definir sus politicas de backup



https://aws.amazon.com/compliance/shared-responsibility-model/
