# Errores comunes en Providers

Si el proceso de creación de un Provider termina en *Error*, generalmente se debe a un problema de conexión con la cuenta de AWS.

Encontrarás el estado de *Error* en el listado de Providers.

A continuación, se describen los casos más comunes y cómo solucionarlos.

### 1. El ID de la cuenta configurada no tiene acceso Root

En este caso, al hacer clic en el botón *Fix*, serás redirigido nuevamente a AWS.

Asegúrate de haber iniciado sesión como usuario Root.

### 2. Se alcanzó el número máximo de cuentas de AWS

AWS tiene un límite de cuentas que puede impedir la creación de nuevas.

Antes de volver a intentar el proceso, aumenta ese límite. De lo contrario, el proceso fallará nuevamente.

<Zoom overlayBgColorEnd="rgba(255, 255, 255, 0.8)">
  <img
    src="/img/provider/providererroracclimit.png"

</Zoom>

### 3. Otros errores

Pueden surgir otros problemas que generalmente se resuelven ejecutando nuevamente la conexión con AWS.

Si el error persiste después de intentar eliminar el Provider y crear uno nuevo, no dudes en reportarnos el problema.
